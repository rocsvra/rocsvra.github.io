小程序(21)：分包

发表于 2024-08-23 分类于 前端 ， 小程序

分包指的是把一个完整的小程序项目，按照需求划分为不同的子包，在构建时打包成不同的分包，用户在使用时按需进行加载。

分包优点

  1. 可以先加载主包，也就是主页面，能优化小程序首次启动的下载时间；
  2. 可以把子包分配给不同的开发人员，分别开发；

分包加载规则

小程序启动时，默认下载主包并启动主包主页，tabBar 页面需要放到主包中；
用户进入分包页面，客户端会把对应分包下载下来，下载完成后再进行展示；

分包体积

主包 + 所有分包 <= 16M；
单个主包/分包 <= 2M；

分包配置

详情 -> 基本信息 -> 本地代码，可以查看分包大小情况。

打包原则

按 subpackages 的配置进行分包，subpackages 之外的目录将被打包到主包中；
最外层的 pages 打包到主包中；
tabBar 页面必须在主包内；
分包不能互相嵌套。

引用原则

主包无法引用分包内的私有资源；
分包之间不能相互引用私有资源；
分包可以引用主包内的公共资源。

独立分包

一般情况下，默认先进入主包，通过主包启动小程序，启动成功后，可以从主包进入分包的页面；
独立分包可以在不打开主包情况下，启动小程序；
独立分包本质上也是分包，但它比较特殊，可以独立于主包和其他分包，单独运行；
一个小程序中可以有多个独立分包。

注意：

主包无法引用独立分包内的私有资源；
独立分包之间，不能相互引用私有资源；
独立分包和普通分包之间，不能相互引用私有资源；
独立分包中不能引用主包内的公共资源。

分包预下载

进入小程序某个页面时，由框架自动预下载可能需要的分包，从而提升进入后续分包页面时的启动速度。

属性 说明
packages 可通过 root 或 name 指定预下载哪些分包
network all（不限网络）和 wifi （仅 wifi 模式下进行预下载），默认值为 wifi

分包预下载限制

共同的预下载大小限额 2M

左边的总体积是 3M ，不允许下载
