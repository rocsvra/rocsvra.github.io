Supervisor简介

发表于 2024-04-17 更新于 2024-04-25 分类于 DevOps

Supervisor是一款进程管理工具。

Supervisor简介

supervisor由Python开发，允许用户在类unix操作系统上控制多个进程；
supervisor能将一个普通命令行进程，变为后台daemon(守护进程)，并监控进程状态；
supervisor是所有相关进程的父进程，以子进程的PID来进行管理，子进程异常退出时，supervisor会收到相应的信号，进而自动重启子进程;

supervisor主要包含以下四个部分:

  1. supervisord：运行Supervisor时，会启动的一个进程，负责管理配置的子进程，包括重启崩溃或异常退出的子进程，同时也响应来自客户端的请求。
  2. supervisorctl：Supervisord服务的命令管理工具，用来执行stop、start、restart 等命令，管理子进程。
  3. Web Server：通过web界面查看和控制进程状态。
  4. XML-RPC Interface: 服务于web UI的同一个HTTP服务器提供一个XML-RPC接口，可以用来询问和控制管理程序及其运行的程序。
