Docker(3)：常用命令

发表于 2021-06-14 更新于 2021-06-15 分类于 Docker

Docker系列3——

docker [command] –help

查看使用方法，例入：docker pull –help

docker search [REPOSITORY]

从Docker Hub查找镜像

NAME: 镜像仓库源名称
DESCRIPTION: 镜像描述
OFFICIAL: 是否 docker 官方发布
stars: 类似 Github 里面的 star
AUTOMATED: 自动构建

docker images

查看本机镜像

REPOSITORY：镜像仓库源
TAG：镜像标签，同意仓库源可以有多个TAG，代表不同版本，默认latest
IMAGE ID：镜像ID
CREATED：镜像创建时间
SIZE：镜像大小

docker pull [REPOSITORY]:[TAG]

获取镜像，如果本机使用一个不存在的镜像，Docker会自动下载

docker rmi

删除镜像

docker commit

提交副本容器，获得新镜像

创建一个容器 docker run
更新容器 apt-get update
退出容器 exit
提交容器副本 docker commit
查看新镜像 docker images

docker build [options] [path]

创建新镜像，需要创建Dockerfile

-t 创建的目标镜像名称
. Dockerfile 文件所在目录

docker tag [container id]

设置镜像标签

设置id为860c279d2fec的镜像，标签为dev

docker run

启动容器

-d 后台运行
-P 容器内部端口随机映射到主机端口
-p 容器内部端口绑定到指定主机端口
–rm 容器退出自动清理容器内部的文件系统
-h 设定容器的主机名，它会被写到容器内的 /etc/hostname 和 /etc/hosts

docker start [container id]

启动一个已停止的容器

docker port

端口绑定情况

docker ps

查看运行的容器

CONTAINER ID: 容器 ID
IMAGE: 使用的镜像
COMMAND: 启动容器时运行的命令
CREATED: 容器的创建时间
STATUS: 容器状态(created | restarting | running或Up removing | paused | exited | dead)
PORTS: 容器的端口信息和使用的连接类型（tcp\udp）
NAMES: 容器名称

docker logs [container id]

获取日志

docker stop [container id]

停止容器

docker restart [container id]

重启容器

docker attach [container id]

进入容器，退出容器终端，会导致容器的停止

docker exec

进入容器，推荐使用，退出容器终端不会导致容器的停止

docker export

导出容器快照

docker import

导入容器快照

docker rm

删除容器
