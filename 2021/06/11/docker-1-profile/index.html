Docker(1)：简介

发表于 2021-06-11 更新于 2023-09-11 分类于 DevOps ， docker

Docker简介

基于Go语言，遵从Apache2.0协议，是一款开源的应用容器引擎
容器沙箱机制，相互间不会有任何接口，性能开销极低
分CE社区版，EE企业版

Docker基本概念

Docker有3个基本概念，镜像、容器、仓库
容器是镜像的实体，类似于面向对象编程中的对象和类
容器可以创建、启动、停止、删除、暂停

Docker架构

Docker使用C/S架构模式，使用远程API管理和创建Docker容器

概念 说明
镜像(iamge) 创建 Docker 容器的模板
容器(container) 独立运行的一个或一组应用，是镜像运行时的实体
客户端(client) 客户端使用命令行或工具，与Docker守护进程通信
仓库(repositry) 用来保存镜像，Docker Hub提供了庞大的镜像集合
主机(host) 物理或虚拟机，用于执行Docker守护进程和容器
Docker Regsitry 一个Registry包含多个仓库；一个仓库包含多个标签；每个标签对应一个镜像
镜像版本格式：[仓库名]:[标签]
默认标签：latest
Docker Machine 如：VirtualBox、 Digital Ocean、Microsoft Azure

Docker Machine

可以在虚拟主机上安装Docker的工具，并使用docker-machine命令管理主机
可以集中管理所有Docker主机

Docker Swarm

Docker 的集群管理工具
