WPF（11）：命令

发表于 2023-05-12 更新于 2023-09-11 分类于 教程 ， WPF

命令可以约束代码，约束步骤逻辑。

命令系统的基本元素

元素 名称 说明
Command 命令 实现了ICommand接口的类，如 RoutedCommand
Command Source 命令源 实现了ICommandSource接口，如Button、MenuItem、ListBoxItem
Command Target 命令目标 实现了IInputElement接口
Command Binding 命令关联 负责关联外围逻辑和命令，如执行前判断、执行后的后续工作

使用步骤

1.创建命令类，实现ICommand接口的类，如：RoutedCommand类；
2.声明命令实例；
3.指定命令源；
4.指定命令目标；
5.设置命令关联；

无须判断按钮是否可用；
RoutedCommand与业务逻辑无关，它的Excute方法只负责让命令目标激发Excuted，则调用cb_Executed；
CanExcuted事件激发频率较高，建议处理完(e.Handled = true;)，避免降低性能；
CommandBinding要设置在外围空间上，不然无法捕捉CanExecute和Executed等路由事件；

WPF命令库

ApplicationCommands
ComponentCommands
NavigationCommands
MediaCommands
EditingCommands

这些都是静态类，命令用这些类的静态只读属性以单例模式暴露出来；
如果程序中用到Open、Save、Play、Stop等标准命令，无须自己声明，可以直接使用；

命令参数（CommandPrameter）

命令源实现了ICommandSource，ICommandSource有一个属性是CommandPrameter。

命令+Binding

Button 关联的命令可能根据某些条件产生改变；

ICommand 和 RoutedCommand

ICommand

只包含两个方法和一个事件

Execute，命令执行；
CanExecute，执行前探知命令是否可执行；
CanExecutedChanged，命令可执行状态发生改变时，可激发此事件通知其他对象；

RoutedCommand

实现了ICommand，并未向Execute和CanExecute方法中添加任何逻辑，是通用的，和业务逻辑无关。

当一个命令到达目标后，具体执行Copy还是Cut（业务逻辑），不是由命令决定，而是外围CommandBinding捕获到命令目标，受命令激发发送路由事件后，在Executed事件处理器中完成。换而言之，CommandBinding捕捉到Copy命令执行清除操作也是可以的。
