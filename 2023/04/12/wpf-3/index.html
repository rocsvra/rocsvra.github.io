WPF（3）：x名称空间

发表于 2023-04-12 更新于 2023-09-11 分类于 教程 ， WPF

x是XAML的首字母，x名称空间也称为“XAML名称空间”。x名称空间的成员(如x:Class、x:Name)，用来引导XAML编译器把XAML代码编译成CLR代码。

XAML语言被编译器解析编译，形成微软中间语言存储在程序集中，解析和编译过程中，需要告诉编译器信息，如XAML编译结果和哪个C#代码编译结果合并，XAML声明的元素的访问级别（public、private）等，这些与XAML编译器沟通的工具存在x名称空间中；

以下是x名称空间包含的工具——

名称 种类
x:Array 标记扩展
x:Null 标记扩展
x:Static 标记扩展
x:Type 标记扩展
x:Class Attribute
x:ClassModifier Attribute
x:FieldModifier Attribute
x:Key Attribute
x:Name Attribute
x:Shared Attribute
x:Subclass Attribute
x:TypeArguments Attribute
x:Uid Attribute
x:Code XAML指令元素
x:XData XAML 指令元素

Attribute

Attribute是语言层面的东西，是给编译器看的；
Property是面向对象层面的东西，是给编程逻辑用的；
一个XAML标签的Attribute里，大部分对应着对象的Property；

x:Class

告诉XAML编译器将XAML标签的编译结果和后台代码指定类合并。

只用于根节点；
前后类型保持一致；
类声明时必须使用partial关键字；

x:ClassModifier

告诉编译器由标签编译生成的类的访问控制级别。

必须有 x:Class ；
前后访问控制级别保持一致；

x:Name

告诉编译器，当标签带有x:Name时，除了为这个标签生成实例外，还要为这个实例声明一个引用变量，变量名就是x:Name的值；
将XAML对应对象的Name属性（如果有），也设为x:Name的值，并把这个值注册到UI树上，方便查找；

可以通过引用者的层级关系找到相应控件；

XAML代码中，Name属性定义在FrameworkElement类中，这个类是WPF控件的基类，所有WPF控件都有这个属性；
当一个元素中具有Name属性时，Name <=> x:Name，两者等价但不能同时使用；
当一个元素中没有Name属性时，只能用x:Name；
一般来说，全部使用x:Name，可以增强代码的统一性和可读性；

x:FieldModifier

使用x:Name后，标签对应实例有了引用变量，这些引用变量都是类的字段；
默认情况下，这些字段的访问级别是internal；
跨程序集访问窗体元素时，就要改变访问控件的访问级别；

x:Key

需要重复使用的XAML内容（如Style、Template、动画等）都需要放在资源(Resources)里，x:Key 可以在资源上贴上用于检索的索引；

x:Shared

x:Shared要和x:Key配合使用；
当x:Shared为true，每次检索到的是同一个对象，默认 x:Shared=”true”；
当x:Shared为false，每次检索到的都是这个对象的新副本；

标记扩展

标记扩展（Markup Extension）实际上就是一些MarkupExtension类的直接或间接派生类；
x名称空间内也包含有一些这样的类；

x:Type

XAML中表达某个数据类型时使用 x:Type标记扩展；

x:Null

显式对属性赋空值；

标签式声明代码不够简洁，使用转义字符串声明；

x:Array

x:Array 通过ArrayExtension的Items属性指向一个ArrayList实例，这个Items属性是只读的，只能使用标签声明语法初始化；

x:Static

使用static成员；

指令元素

x:Code

x:XData

在XAML中声明一个带有数据的XmlDataProvider实例。
