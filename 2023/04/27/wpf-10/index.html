WPF（10）：路由事件

发表于 2023-04-27 更新于 2023-05-12 分类于 WPF

事件系统在WPF中得到了升级，进化为路由事件（Routed Event），并在其基础上衍生出命令传递机制。

UI组件树

WPF事件的路由环境是UI组件树，WPF有两种树：

逻辑树（Logical Tree），由布局组件和控件构成；
可视元素树（Visual Tree）

将进度条改造成温度计，需要在其内部添加树形结构；
如果把Logical Tree 延伸至Template组件级别，得到的就是Visual Tree；

在Logical Tree查找元素，可以使用LogicalTreeHelper类的静态方法：

BringIntoView：选定元素带进可视区域，常用于可滚动视图；
FindLogicalNode：根据Name属性值查找元素，包括子级树上的元素；
GetChildren：获取所有直接子级元素；
GetParent：获取直接父级元素；

在Visual Tree上导航或查找元素，可借助VisualTreeHelper类的静态方法；

CLR直接事件模型

事件实际上是一个使用event关键字修饰的委托类型成员变量，事件处理器则是一个函数；
CLR直接事件模型中，事件的拥有者就是消息的发送者（sender）；
每对消息都是“发送-响应”关系，必须建立显式的点对点订阅关系；
事件宿主必须能直接访问事件的响应者，不然无法建立订阅关系；

路由事件

直接事件激发时，发送者直接将消息通过事件订阅交送给事件响应者，事件响应者使用事件处理器对事件作出响应；
路由事件的事件拥有者和事件响应者之间没有显式的订阅关系，事件拥有者只负责激发事件，事件响应者则安装有事件侦听器，对某类事件进行侦听；当事件传递至事件响应者时，事件响应者使用事件处理器响应事件，并决定是否可以继续传递；

单击buttonLeft，Button.Click事件沿着 buttonLeft -> canvasLeft -> gridA -> gridRoot -> Window 这条路线向上传递；

自定义路由事件

创建步骤：
1.声明并注册路由事件；
2.为路由事件添加CLR事件包装；
3.创建可以激发路由事件的方法；

路由事件案例：

Bubble策略，TimeButton -> StackPanel -> Grid -> Grid -> Grid -> Window
Tunnel策略，Window -> Grid -> Grid -> Grid -> StackPanel -> TimeButton
事件经过gird_2被处理，不再传递

RoutedEventArgs的Source和OriginalSource

Source：LogicalTree上的消息源
OriginalSource：VisualTree上的消息源

附加事件

附加事件就是路由事件，但路由事件的宿主都是些拥有可视化实体的界面元素，附加事件的宿主没有界面渲染功能。

升级版：

附加事件宿主不是UIElement派生类，无法出现在UI结点上，附加事件的激发是借助UI元素实现的，激发事件需要附加在某个具有RaiseEvent方法的对象上；
